# スマホ ⇔ パソコン データ共有のセットアップ手順

このアプリのデータ（患者・症例・評価・コメントなど）は、通常は各端末のブラウザ内にしか保存されません。
以下の設定を行うと、**クラウド（Supabase）経由でスマホとパソコンのデータが自動的に共有**されるようになります。

## 仕組み

- アプリを開いたとき → クラウドから最新データを自動取得
- データを保存したとき → 約1.5秒後にクラウドへ自動送信
- サイドバーの手動同期ボタンでいつでも即時送信可能

> **注意**: 端末内に保存した動画ファイル本体は共有されません（データ量が大きいため）。
> Vercel Blob にアップロードした動画は URL で共有されるため、どの端末からも再生できます。

---

## 手順 1: Supabase の準備（無料・約5分）

1. [https://supabase.com](https://supabase.com) にアクセスし、アカウントを作成（Google アカウントでOK）
2. 「New Project」で新しいプロジェクトを作成
   - Name: `yuuki-rehab` など好きな名前
   - Database Password: 自動生成のままでOK（控えておく）
   - Region: `Northeast Asia (Tokyo)` を選択
3. プロジェクトが起動したら、左メニューの **SQL Editor** を開く
4. このリポジトリの `supabase-setup.sql` の中身を貼り付けて **Run** を実行
5. 左メニューの **Project Settings → API** を開き、次の2つを控える
   - **Project URL**（`https://xxxx.supabase.co` の形式）
   - **service_role キー**（`Legacy API Keys` タブ内。⚠️ 絶対に他人に見せないこと）

## 手順 2: 合言葉（同期シークレット）を決める

スマホとパソコンが「同じ院のデータ」と確認し合うための合言葉です。
推測されにくい長いランダム文字列にしてください（例: パスワード生成ツールで32文字）。

## 手順 3: 環境変数を設定する

### Vercel にデプロイしている場合

1. Vercel のダッシュボード → 対象プロジェクト → **Settings → Environment Variables**
2. 以下の4つを追加：

| 名前 | 値 |
|------|-----|
| `SUPABASE_URL` | 手順1で控えた Project URL |
| `SUPABASE_SERVICE_ROLE_KEY` | 手順1で控えた service_role キー |
| `SYNC_SECRET` | 手順2で決めた合言葉 |
| `NEXT_PUBLIC_SYNC_SECRET` | `SYNC_SECRET` と**同じ値** |

3. **再デプロイ**する（Deployments → 最新のデプロイ → Redeploy）。
   `NEXT_PUBLIC_〜` はビルド時に埋め込まれるため、再デプロイしないと反映されません。

### パソコンでローカル起動する場合

プロジェクト直下に `.env.local` を作成（既にあれば追記）：

```
SUPABASE_URL=https://xxxx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOi...
SYNC_SECRET=（合言葉）
NEXT_PUBLIC_SYNC_SECRET=（合言葉と同じ値）
```

その後 `npm run dev` を起動し直してください。

## 手順 4: 動作確認

1. **スマホ**でアプリを開き、患者や症例を1件登録・編集する
2. 数秒待つ（自動同期）か、サイドバーの同期ボタンを押す
3. **パソコン**のブラウザでアプリを開いてログインする
4. スマホで登録した内容が表示されれば成功 🎉
   （すでに開いていた場合はページを再読み込みしてください）

## よくある質問

**Q. 同時に2台で編集したらどうなる？**
最後に保存した端末の内容で上書きされます。同じ患者を同時に編集するのは避けてください。

**Q. 同期されていないみたい**
- 4つの環境変数がすべて設定されているか確認（特に `NEXT_PUBLIC_SYNC_SECRET`）
- Vercel の場合は設定後に再デプロイしたか確認
- 合言葉が全端末（＝同じデプロイ）で一致しているか確認

**Q. セキュリティは大丈夫？**
- service_role キーはサーバー側（API Route）でのみ使用され、ブラウザには渡りません
- 同期 API は合言葉（`SYNC_SECRET`）が一致しないと 401 エラーで拒否します
- 合言葉は十分に長いランダム文字列にしてください
