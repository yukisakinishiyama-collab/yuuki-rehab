# 13_KPI_Definition — KPI 定義書 v1.0

最終更新：2026-07-12 ／ 版：v1.0 ／ 承認状態：CEO 承認待ち（定義の承認後に固定）
管理：BI AI ／ 定義の変更は CEO 承認事項。変更時は版を上げ、過去数値との非連続を明示する。

## 共通ルール

- 目標値はすべて**CEO入力待ち**。現状値の入力後、3か月の実測を基準に設定することを推奨
- 「取得元」が未整備の KPI は、整備までの間**月次手入力**（`docs/bi/kpi-input-template.md`）で運用する
- 欠測は 0 ではなく「欠測」と記録する

## A. 売上・経営

### A-1 月売上
| 項目 | 定義 |
|---|---|
| 目的 | 経営の総合指標。全施策の最終的な妥当性確認 |
| 分子／分母 | 当月総売上（保険＋自費＋物販）／なし |
| 計算式 | 保険売上＋自費売上＋物販売上 |
| データ取得元 | レセコン・会計記録（将来：CRM） |
| 入力頻度 | 月次 |
| 目標値 | CEO入力待ち |
| 注意点 | 保険分は請求ベースか入金ベースかを初回に固定する（入金は2か月遅れるため混在禁止） |
| AI が分析できること | 季節性・施策との前後比較・チャネル別新患との相関 |
| CEO が判断すべきこと | 目標値・投資判断（施策拡大/縮小） |

### A-2 日別売上
| 項目 | 定義 |
|---|---|
| 目的 | 曜日・時間帯の傾向把握、スタッフ配置と営業時間の最適化材料 |
| 分子／分母 | 日次総売上／なし |
| 計算式 | 当日会計合計 |
| データ取得元 | 日計表 |
| 入力頻度 | 日次（集計報告は月次） |
| 目標値 | CEO入力待ち |
| 注意点 | 祝日・学校行事・大会シーズンをメモ欄に残す（変動要因の記録がないと分析不能） |
| AI が分析できること | 曜日別・時間帯別パターン、繁閑の予測 |
| CEO が判断すべきこと | 営業時間・予約枠の変更 |

### A-3 自費率
| 項目 | 定義 |
|---|---|
| 目的 | 収益構造の健全性（保険依存度）の把握 |
| 分子／分母 | 当月自費売上／当月総売上 |
| 計算式 | 自費売上 ÷ 総売上 × 100 |
| データ取得元 | レセコン・会計記録 |
| 入力頻度 | 月次 |
| 目標値 | CEO入力待ち |
| 注意点 | 自費への誘導が患者利益を損なわないこと（01_Mission のバリュー優先）。比率目標を現場ノルマ化しない |
| AI が分析できること | 自費メニュー別の内訳推移、新患層との関係 |
| CEO が判断すべきこと | 自費メニューの価格・構成 |

### A-4 保険施術比率
| 項目 | 定義 |
|---|---|
| 目的 | 療養費の適正運用の監視（急性外傷中心の院方針との整合確認） |
| 分子／分母 | 当月保険施術売上／当月総売上 |
| 計算式 | 保険売上 ÷ 総売上 × 100 |
| データ取得元 | レセコン |
| 入力頻度 | 月次 |
| 目標値 | CEO入力待ち |
| 注意点 | 柔整療養費は急性・亜急性の外傷に限定。比率の高低自体より「適用が適正か」が本質。長期継続例は個別に妥当性を確認 |
| AI が分析できること | 部位別・継続期間別の傾向、外傷比率との整合 |
| CEO が判断すべきこと | 長期継続例の扱い・返戻リスクへの対応 |

### A-5 物理療法利用率
| 項目 | 定義 |
|---|---|
| 目的 | 設備稼働と施術構成の把握 |
| 分子／分母 | 物理療法を実施した施術件数／総施術件数 |
| 計算式 | 物療実施件数 ÷ 総施術件数 × 100 |
| データ取得元 | 施術記録（将来：症例管理 AI） |
| 入力頻度 | 月次 |
| 目標値 | CEO入力待ち |
| 注意点 | 利用率の最大化が目的ではない（適応に基づく実施が前提）。機器別に分けると設備投資判断に使える |
| AI が分析できること | 機器別稼働・症状別の利用パターン |
| CEO が判断すべきこと | 機器の更新・追加投資 |

## B. 患者フロー

### B-1 新患数
| 項目 | 定義 |
|---|---|
| 目的 | 集患施策（MEO・SNS・紹介）の最重要アウトカム |
| 分子／分母 | 当月初来院の患者数／なし |
| 計算式 | 初診会計数（過去に来院歴のない患者） |
| データ取得元 | レセコン・予約台帳。**初診時に「来院経路」を必ず聴取**（GBP/Instagram/紹介/通りがかり/その他） |
| 入力頻度 | 月次（経路内訳つき） |
| 目標値 | CEO入力待ち |
| 注意点 | 来院経路の聴取が抜けると全チャネル分析が成立しない。受付フローに組み込む |
| AI が分析できること | 経路別新患の推移、GBP/SNS 指標との相関、テーマ展開との前後比較 |
| CEO が判断すべきこと | チャネル別のリソース配分 |

### B-2 再診率
| 項目 | 定義 |
|---|---|
| 目的 | 初回体験の質と計画的な通院継続の把握 |
| 分子／分母 | 当月新患のうち30日以内に2回目来院した人数／当月新患数 |
| 計算式 | 30日以内再来院新患 ÷ 新患数 × 100（翌月末に確定） |
| データ取得元 | レセコン・予約台帳 |
| 入力頻度 | 月次（1か月遅れで確定） |
| 注意点 | 症状が軽快して来なくなるのは望ましい離脱。「必要な通院計画に対する継続」かを施術録と併せて解釈する |
| 目標値 | CEO入力待ち |
| AI が分析できること | 初回対応者別・症状別の差、離脱タイミングの分布 |
| CEO が判断すべきこと | 初診フロー・説明資料の改善投資 |

### B-3 LTV（患者生涯価値）
| 項目 | 定義 |
|---|---|
| 目的 | 新患1人の獲得価値の把握（広告投資判断の基礎） |
| 分子／分母 | 期間総売上／期間ユニーク患者数（簡易定義） |
| 計算式 | 直近12か月総売上 ÷ 同期間ユニーク患者数 |
| データ取得元 | レセコン（将来：CRM で患者単位の累計を自動算出） |
| 入力頻度 | 四半期 |
| 目標値 | CEO入力待ち |
| 注意点 | 簡易定義（平均値）から開始し、CRM 導入後に患者単位定義へ移行。定義切替時は非連続を明示 |
| AI が分析できること | 症状カテゴリ別 LTV、紹介患者と検索経由患者の LTV 差 |
| CEO が判断すべきこと | LTV 定義の切替時期、広告投資の上限 |

### B-4 予約率
| 項目 | 定義 |
|---|---|
| 目的 | 通院計画の合意度（次回予約を取って帰る割合） |
| 分子／分母 | 会計時に次回予約を取得した件数／総会計件数 |
| 計算式 | 次回予約取得数 ÷ 会計数 × 100 |
| データ取得元 | 予約台帳 |
| 入力頻度 | 月次 |
| 目標値 | CEO入力待ち |
| 注意点 | 卒業（計画終了）の会計は分母から除くか含めるかを初回に固定 |
| AI が分析できること | 施術者別・症状別の差、再診率との相関 |
| CEO が判断すべきこと | 予約運用ルールの変更 |

### B-5 キャンセル率
| 項目 | 定義 |
|---|---|
| 目的 | 予約枠のロス把握と患者エンゲージメントの間接指標 |
| 分子／分母 | キャンセル件数（連絡あり＋無断）／予約総数 |
| 計算式 | キャンセル数 ÷ 予約総数 × 100 |
| データ取得元 | 予約台帳 |
| 入力頻度 | 月次 |
| 目標値 | CEO入力待ち |
| 注意点 | 連絡ありと無断を必ず分けて記録（対策が異なる） |
| AI が分析できること | 曜日・時間帯・患者層別のパターン、リマインド導入の効果測定 |
| CEO が判断すべきこと | キャンセルポリシーの設定 |

### B-6 無断キャンセル率
| 項目 | 定義 |
|---|---|
| 目的 | 最も損失の大きいロスの監視。リマインド自動化（L2）の効果指標 |
| 分子／分母 | 無断キャンセル件数／予約総数 |
| 計算式 | 無断キャンセル数 ÷ 予約総数 × 100 |
| データ取得元 | 予約台帳 |
| 入力頻度 | 月次 |
| 目標値 | CEO入力待ち |
| 注意点 | リマインド送信の有無を記録し、効果検証できる形にする |
| AI が分析できること | リマインド有無での比較、繰り返し無断キャンセル患者の検出（院内対応用・匿名集計外の個別対応は L3） |
| CEO が判断すべきこと | 繰り返し事例への対応方針 |

### B-7 紹介率
| 項目 | 定義 |
|---|---|
| 目的 | 信頼形成の成果指標（バリュー「信頼形成」の代表 KPI） |
| 分子／分母 | 紹介経由の新患数／新患総数 |
| 計算式 | 紹介新患 ÷ 新患総数 × 100 |
| データ取得元 | 初診時の来院経路聴取 |
| 入力頻度 | 月次 |
| 目標値 | CEO入力待ち |
| 注意点 | 「家族・友人」「医療機関からの紹介」を分けて記録（後者は連携の成果指標として別掲） |
| AI が分析できること | 紹介元の傾向、紹介カード施策（14_Offline）の効果測定 |
| CEO が判断すべきこと | 紹介カード等の施策実施 |

## C. MEO・GBP

### C-1 Google 口コミ数 ／ C-2 口コミ平均
| 項目 | 定義 |
|---|---|
| 目的 | ローカル検索での選ばれやすさ（チャネル優先度②）の中核指標 |
| 分子／分母 | 累計口コミ件数（月次で新規数も記録）／平均は星の単純平均 |
| データ取得元 | GBP 管理画面 |
| 入力頻度 | 月次 |
| 目標値 | CEO入力待ち |
| 注意点 | **対価提供・選別依頼は禁止**（Google ポリシー・ステマ規制）。返信率も併記推奨 |
| AI が分析できること | 増加ペース、低評価の内容分類、返信の効果 |
| CEO が判断すべきこと | 口コミ依頼導線の運用開始、低評価への対応 |

### C-3 GBP 表示回数 ／ C-4 経路検索数 ／ C-5 電話タップ数 ／ C-6 Web サイトクリック数
| 項目 | 定義 |
|---|---|
| 目的 | MEO ファネルの計測：表示（認知）→ 経路・電話・クリック（行動） |
| 定義 | 表示回数＝検索・マップ合算表示。経路＝ルート検索タップ。電話＝通話ボタンタップ。クリック＝サイトリンククリック |
| データ取得元 | GBP パフォーマンスレポート |
| 入力頻度 | 月次 |
| 目標値 | CEO入力待ち |
| 注意点 | Google 側の仕様変更で数値が非連続になることがある（変更月を記録）。表示→行動の転換率も算出する |
| AI が分析できること | 投稿・写真追加・情報更新と表示/行動の前後比較、検索語レポートとの突合 |
| CEO が判断すべきこと | MEO 施策の継続・変更 |

## D. SNS（Instagram）

### D-1 リーチ ／ D-2 保存率 ／ D-3 プロフィールアクセス
| 項目 | 定義 |
|---|---|
| 目的 | 認知（リーチ）・コンテンツ価値（保存率）・関心（プロフィールアクセス）の3段計測 |
| 定義 | リーチ＝月間合計。保存率＝投稿ごとの保存数÷リーチ（月次は投稿平均）。プロフィールアクセス＝月間合計 |
| データ取得元 | Instagram インサイト（SNS AI 経由） |
| 入力頻度 | 月次 |
| 目標値 | CEO入力待ち |
| 注意点 | フォロワー数よりも保存率を重視（保存＝保護者の「後で見返す」行動で、当院ターゲットの質指標）。投稿数が変わる月は1投稿あたりで比較 |
| AI が分析できること | テーマ別・形式別（カルーセル/リール）の保存率比較、投稿時間の最適化 |
| CEO が判断すべきこと | 投稿配分（カルーセル:リール:ストーリーズ）の変更 |

## E. CRM

### E-1 LINE 登録数
| 項目 | 定義 |
|---|---|
| 目的 | 再来院・リマインド・患者教育の直接チャネルの規模 |
| 分子／分母 | 累計友だち数（月次で新規・ブロック数も記録）／なし |
| データ取得元 | LINE 公式アカウント管理画面 |
| 入力頻度 | 月次 |
| 目標値 | CEO入力待ち |
| 注意点 | ブロック率が高い場合は配信内容・頻度を見直す。配信は L2（承認後）運用 |
| AI が分析できること | 登録経路（院内/SNS/GBP）別の伸び、配信とブロックの関係 |
| CEO が判断すべきこと | 配信頻度・内容方針 |
